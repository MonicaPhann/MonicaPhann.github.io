Project Name: Mechanical Engineering Portfolio:

Project Overview

  • Describe your application and its purpose:

    This project is a personal portfolio website for a mechanical engineering student. The purpose of the website is to showcase the student’s academic background, technical skills, engineering projects, certifications, and resume to prospective employers, graduate schools, and collaborators. It serves as a digital resume, providing a professional online presence that highlights the student’s competencies and career goals.

  • Intented users of the website:
    • Potential employers in engineering or manufacturing industries.
    • Graduate program admission officers.
    • Professors or academic mentors.
    • Networking contacts and peers.
    • Recruiters and HR professionals.
  • Overview of the content of the website:
    • Home Page: Brief welcome and professional summary.
    • About Me: Educational background, goals, and skills.
    • Projects: Detailed descriptions and media from enineering projects.
    • Resume: Downloadable resume and career highlights
    • Contact: Form for inquiries and links to social/professional platforms.

Dynamic Functionality on the Website:

This portfolio website will incorporate several interactive and dynamic features using JavaScript and jQuery/jQuery UI to improve user experience, visual engagement, and usability. These interactive elements help present the student’s skills and projects in a professional and appealing way.

  • 1. Contact Form with Real-Time Validation:
    • Page: Contact Page
    • Functionality: The form will include real-time validation using JavaScript. As the user types, it will validate the name (minimum 2 characters), email (valid format), and message (minimum 10 characters). If invalid, it will show an error immediately without submitting.
    • Why: Ensures data integrity and improves user experience with immediate feedback
    • Example Site: getbootstrap
  • 2. Project Gallery with Expandable Details:
    • Page: Projects Page
    • Functionality: Projects will be listed with short summaries. Clicking on a project will either expand (accordion) or open a modal to display full details, images, and external links (like GitHub). This will be implemented using jQuery UI or a custom modal.
    • Why: Helps organize multiple projects cleanly without overwhelming the page; gives users control over what they want to explore.
    • Example Site: behance
  • 3. Smooth Scroll Navigation:
    • Page: All Pages (navigation)
    • Functionality: Clicking on nav links will smoothly scroll the page to the respective section using JavaScript/jQuery.
    • Why: Improves UX by making navigation feel polished and responsive.
    • Example Site: css-tricks
  • 4. jQuery UI Tabs for Skills and Certifications:
    • Page: About Me Page
    • Functionality: Skills and certifications will be organized in jQuery UI tabs, allowing users to switch between categories like “Software,” “Mechanical Tools,” and “Certifications.”
    • Why: Presents technical content in a compact and user-friendly format.
    • Example Site: jqueryui
  • 5. AJAX Form Submission:
    • Page: Contact Page
    • Functionality: When the contact form is submitted, it will use AJAX to send the data to a server or mock endpoint without reloading the page. A success or error message will be displayed dynamically.
    • Why: Ensures a smoother user experience without disrupting navigation.
    • Example Site: w3schools
  • 6. Image Hover Effects and Tooltips:
    • Page: Projects Page
    • Functionality: On hovering over a project image, a short description or project name will appear via tooltip or overlay.
    • Why: Adds interactivity and gives a quick idea about the project before clicking.
    • Example Site: tympanus
